doxygen是一个代码注释规范,可以注释多个语言的代码,通过对应的软件可以生成对应的文档,下面是doxygen的简单学习

1. doxygen的安装

安装doxygen以及对应的依赖graphviz

sudo apt-get install doxygen
sudo apt-get install graphviz

2. doxygen的使用规范

clion对doxygen有默认的操作,可以直接进行doxyge n风格的注释:doxygen+clion

键入以下符号之一：///(C风格)，//!(C风格)，/**(Javadoc风格)或/*!(Qt风格)然后按Enter,即可自动生成对应的风格的doxygen注释,一般地,下面按照mysql的默认的javadoc风格,即使用/**:

/**
 * 
 * @param a 
 * @param b 
 * @param c 
 * @return 
 */
int sumabc(int a,int b,int c){
  return a+b+c;   
}

如果是clion的话,几个常用的注解以及对应的含义如下:

名字 意义
无 函数/类的详细的描述
@param 形式参数的注解
@return 返回值的注解
@author/@authors 作者(们)
@todo 要做的事情
@brief 简要描述,需要放在最前面
@date 时间

注意:如果希望换行,注释需要空一行,生成的文档才能换行

2.1 我们规定的使用的命令

2.1.1 普通行的注释

单行注释就是普通的注释可以采用以下几个普通的注释:

/**/
//

2.1.2 单行变量/单行typedef的注释

使用doxygen格式对局部变量的定义注释:

int a=0; ///<...some comments...

如果是doxygen,则当鼠标悬浮在对应的变量时候,对应的变量会被识别:

2.1.2 函数的注释

下面是一个实际的例子:

/**@brief 简洁描述
 * 简洁描述继续

 * 详细描述

 * @author cwy
 * @date 2020年8月4日
 * @param a 参数a
 * @param b 参数b
 * @param c 参数c
 * @return 返回a+b+c
 */
int sumabc(int a,int b,int c){
  return a+b+c;
}

int main() {
    int a = 0;///< 这是一个测试的变量:a
    a;
    sumabc(1,1,1);
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

2.1 分组(模块)注释

模块(module)可以把东西分组组织到同一个页中。 这样组中的成员可以是文件，命名空间，类，函数，变量，枚举，typedef,宏定义，以及其它的组。

对于组别,可以有以下几个相关的注解:

名称 含义
@defgroup 定义群组
@ingroup 将函数/类划入某个群组

@defgroup和@ingroup不必非要在一个文件里面,下面是一个@defgroup例子:

/** @defgroup cwy
 * cwy注释的代码
 * */

@defgroup命令有两个参数，第一个参数("cwy")用来唯一标识组，第二个参数("cwy注释的代码")是指明该组在文档中显示的标题。

当需要使用分组的时候,可以对要使用的对象使用@ingroup注解:

/**
 * @ingroup 蔡文玉
 * 测试类
 */
class Test1
{
public:
    int a;///< 属性a
    /**
     * 详细说明
     * @param a 参数a
     * @param b 参数b
     * @return 返回值a+b
     */
    int add(int a,int b){
        return a+b;
    }
};

特殊地,单行也可以进行注解:

int x;///<x坐标 @ingroup cwy

最后生成的文档在module分页下.

注意:@defgroup和@ingroup不必非要在一个文件内

3. doxygen注释生成过程操作实例

3.1 准备工作

  1. 安装doxygen

    详见第一节

  2. 在项目的根目录打开命令行:

  3. 执行:

    cwy@mysql:~/桌面/test$ doxygen -g -s


Configuration file `Doxyfile' created.

Now edit the configuration file and enter

  doxygen Doxyfile

to generate the documentation for your project

  4. 编辑根目录生成的配置文件中的以下几项:

    #给出所有文档的输出目录
OUTPUT_DIRECTORY       = doc(文档文件输出文件夹)
#设置使用的语言
OUTPUT_LANGUAGE 	   = Chinese
#生成latex文档
GENERATE_LATEX         = NO

    其他的选项可以自己选择,如生成latex文件:

    #生成latex文档
GENERATE_LATEX         = YES

    生成rtf文件:

    #生成rtf文档
GENERATE_RTF           = YES

  5. 执行

    在根目录输入命令:

    cwy@mysql:~/桌面/test$ doxygen

    即可得到对应文件夹以及相关文件:
